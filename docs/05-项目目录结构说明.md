# 项目目录结构说明

---

* **`web_project/`**：根项目目录，包含整个 SQL 智能教学系统的所有组成部分。

  * **`sql-edu-backend/`**：后端服务目录，基于 FastAPI 框架开发。
    * **`main.py`**：程序总入口，负责启动整个 API 服务。
    * **`dependencies.py`**：放置 `get_session` 等依赖项。
    * **`requirements.txt`**：列出后端所需的所有 Python 依赖，用于环境搭建和部署。
    * **`alembic.ini`**：Alembic 数据库迁移工具的配置文件。
    * **`.env`**：环境变量配置，存放 DB_URL、AI_API_KEY、JWT_SECRET_KEY 等敏感或环境相关配置。
    * **`.gitignore`**：Git 忽略规则，指定不被版本控制跟踪的文件和目录。
    * **`routers/`**：使用 APIRouter 分层，包含：
      * `auth_router.py`：用户认证、登录、注册相关路由。
      * `ai.py`：AI 对话、辅导相关路由。
      * `question.py`：题目管理、提交判题相关路由。
    * **`core/`**：放置 AI 服务与业务的核心逻辑，包含：
      * `ai_service.py`：AI 服务核心实现。
      * `ai_question_generator.py`：按知识点 AI 生成题目。
      * `sql_judge.py`：判题逻辑。
      * `judge_setup.py`：判题环境初始化。
      * `difficulty_service.py`：难度评估。
      * `experience_service.py`：经验值与等级计算。
      * `sql_knowledge_points.py`：知识点管理等。
    * **`models/`**：放置 SQLAlchemy 的数据库模型（User、Question、Submission、Chat、Auth 等）。
    * **`schemas/`**：放置 Pydantic 模型（UserCreate、UserOut、QuestionCreate、SubmissionIn 等请求/响应模型）。
    * **`repository/`**：专门存放操作数据库的函数（如 `get_user_by_id`、`get_question_by_id` 等）。
    * **`settings/`**：存放配置文件（如 config.py 中的 DB_URL、AI_API_KEY 等）。
    * **`alembic/`**：数据库迁移工具 Alembic 的配置和脚本目录。
      * `env.py`：Alembic 运行环境配置。
      * `versions/`：迁移版本脚本（如 initial_schema、add_user_role 等）。
    * **`scripts/`**：辅助脚本（如 init_judge_tables.sql）。
    * **`tests/`**：后端单元测试与集成测试（test_api_integration、test_sql_judge、test_scaffolding 等）。

  * **`sql-edu-frontend/`**：前端应用目录，基于 uni-app + Vue 3 开发，支持多端发布。
    * **`src/`**：前端项目源代码目录。
      * **`pages/`**：存放各页面组件。
        * `login/index.vue`：登录页。
        * `index/index.vue`：学生练习页。
        * `teacher/index.vue`：教师端管理页。
      * **`api/`**：接口封装，统一管理和调用后端 API（auth.ts、ai.ts、questions.ts）。
      * **`utils/`**：工具函数（request.ts 封装 HTTP 请求、auth.ts 处理认证等）。
      * `App.vue`：根组件。
      * `main.ts`：前端入口。
      * `pages.json`：uni-app 页面路由配置。
      * `manifest.json`：uni-app 应用 manifest 配置。
      * `uni.scss`：全局样式变量。
      * `env.d.ts`：TypeScript 环境类型声明。
    * **`index.html`**：前端应用的 HTML 入口。
    * **`package.json`**：项目元数据与 npm 依赖声明。
    * **`package-lock.json`**：依赖版本锁定文件。
    * **`vite.config.ts`**：Vite 构建工具配置。
    * **`tsconfig.json`**：TypeScript 编译器配置。
    * **`shims-uni.d.ts`**：uni-app 类型声明补丁。
    * **`.npmrc`**：npm 客户端配置。
    * **`.gitignore`**：前端 Git 忽略规则。

  * **`docs/`**：项目文档目录，存放各类说明与指南。
    * `01-各板块功能介绍.md`：各功能模块介绍。
    * `02-从0实现逻辑链.md`：核心逻辑链从零实现说明。
    * `03-前后端功能测试文档.md`：前后端功能测试步骤与用例。
    * `04-从零到运行-环境与安装步骤.md`：环境配置与安装步骤。
    * `05-项目目录结构说明.md`：本文档，项目目录结构说明。
    * `06-上传到GitHub指南.md`：清除敏感信息并上传到 GitHub 的步骤。

  * **`README.md`**：项目简介、快速开始与文档索引。
